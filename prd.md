# [PRD] AI 부트캠프 팀 프로젝트 협업 관리 앱 (TeamFlow)

본 문서는 전공자와 비전공자가 혼합된 대학생 AI 부트캠프 팀을 위한 프로젝트 진행 상황 관리 애플리케이션 **TeamFlow**의 제품 요구사항 정의서(PRD)입니다.

---

## 1. 기획 (Product Planning)

### 📌 서비스 주제와 해결할 문제 정의
*   **서비스 주제**: 비전공자와 전공자(AI/SW 개발자) 간의 협업 장벽을 낮추는 AI 프로젝트 전용 진행 상황 관리 서비스.
*   **해결할 문제**:
    1.  **협업 도구의 높은 진입 장벽 (Git/Jira)**: 전공자 위주의 Git 연동이나 복잡한 Jira 설정은 비전공자(기획, 데이터 수집, 디자인 담당) 팀원들이 진행 상황을 제때 업데이트하지 못하고 소외되는 원인이 됨.
    2.  **개발 태스크와 비개발 태스크의 분리**: AI 프로젝트의 특성상 데이터 수집/정제, 프롬프트 테스트, 발표 자료 준비 등 비개발 태스크와 AI 모델 학습, 웹 서빙 등 개발 태스크가 뒤섞여 있으나 이를 한눈에 관리하기 어려움.
    3.  **의사소통의 불일치**: 전공자끼리만 공유되는 코드 수준의 진행 상황을 비전공자도 직관적으로 이해할 수 있는 브릿지 역할의 도구가 부재함.

### 👥 주 사용자 특성 기술
*   **주 사용자**: 대학생 AI 부트캠프 참가자 (5~7인 규모의 팀원들)
    *   **전공자 (SW/AI 개발 담당)**: GitHub 사용에 익숙하며, 모델 실험 및 웹 애플리케이션 서빙을 담당.
    *   **비전공자 (기획/데이터/PM 담당)**: 코드 작성이 낯설거나 데이터 라벨링, 기획안 작성, 프롬프트 엔지니어링, 디자인을 담당.
*   **사용자 요구사항**:
    *   비전공자도 쉽게 회원가입하고 조작할 수 있는 직관적인 UI.
    *   코드를 만지지 않고도 진행 상황을 손쉽게 기록하고 전공자의 작업 흐름을 한눈에 파악하고 싶어 함.
    *   전공자는 기존 개발 흐름(GitHub PR 등)을 깨지 않고 태스크를 자동 연동하고 싶어 함.

### 🚀 MVP 기능 목록 (4개)
1.  **역할 구분형 실시간 칸반 보드 (Role-based Kanban)**
    *   데이터/기획(비전공 영역)과 모델/개발(전공 영역) 태스크를 시각적으로 분리하거나 필터링하여 볼 수 있는 보드.
    *   실시간 동기화로 다른 팀원의 상태 업데이트가 즉시 반영됨.
2.  **이원화된 작업 완료 시스템 (Hybrid Integration)**
    *   **전공자**: GitHub PR 머지 시 연동된 칸반 카드가 자동으로 `Done` 상태로 변경.
    *   **비전공자**: 웹 UI에서 간편한 원클릭 버튼 및 드래그 앤 드롭으로 태스크 완료 상태 변경 가능.
3.  **AI 기반 태스크 자동 판별 및 등록 시스템 (AI Auto-Tagging)**
    *   사용자가 태스크 제목과 하고자 하는 작업 내용만 입력하여 생성을 요청하면, AI(Gemini)가 텍스트를 심층 분석하여 적절한 역할 분야(기획/데이터/모델/개발) 및 기술적 난이도(상/중/하, 비전공자 수행 가능 여부)를 자동 판별해 태그로 부착하고 데이터베이스에 등록.
    *   이를 통해 비전공자가 전공자의 고난도 개발 태스크나 비개발 직군의 작업 공수를 오판하여 태깅하는 문제를 사전에 예방하고 데이터 신뢰도 확보.
4.  **팀 데일리 랩업 및 요약 (Daily Wrap-up)**
    *   매일 종료 시점에 완료된 태스크들을 AI가 분석하여 비기술직군도 이해하기 쉬운 형태의 '오늘의 프로젝트 요약 보고서'로 자동 변환하여 제공.

### 🔍 레퍼런스 서비스
1.  **Asana (asana.com)**: 직관적이고 시각적인 태스크 흐름 제공으로 비개발 직군도 쉽게 사용 가능한 협업 툴.
2.  **Linear (linear.app)**: 개발 직군이 선호하는 초고속 태스크 연동 및 깃허브 오토메이션 툴.

## 2. 기술 스택 & 개발 환경 (Technology Stack)

### 💻 기술 스택 결정
*   **Frontend / Backend**: Next.js 14+ (App Router, TypeScript)
*   **Styling**: Vanilla CSS (CSS Modules 또는 CSS 변수를 활용해 깔끔하고 화려한 다크 모드 및 마이크로 인터랙션 구현)
*   **State Management**: Zustand
*   **Database ORM**: Prisma 또는 Drizzle ORM (Neon DB와 간편한 스키마 동기화)
*   **Authentication**: NextAuth.js (구글 및 이메일 로그인 연동)
*   **Realtime**: Pusher (무료 채널을 활용한 실시간 칸반 업데이트 동기화)
*   **AI Integration**: OpenRouter API / Gemini API (태스크 난이도 분석 및 요약 리포트 생성용)

### 🗄️ DB 사용 여부 및 종류
*   **사용 여부**: **사용 필수**
*   **종류**: Serverless PostgreSQL (**Neon DB**)
    *   선정 이유: Neon DB는 완전한 서버리스 PostgreSQL 데이터베이스로서 분기(Branching) 기능이 뛰어나 개발/기획 단계에서 분리된 DB 환경을 손쉽게 구축할 수 있으며, Next.js 및 Vercel과의 연동이 매끄럽고 넉넉한 프리 티어를 제공함.

### 🌐 배포 환경 선택
*   **Application / API Hosting**: Vercel (Next.js 호환성 극대화 및 무료 배포)
*   **Database**: Neon DB Cloud (Free Tier)

### 🔌 외부 API / 연동 서비스 목록
*   **GitHub REST API & Webhook**: 전공자용 코드 연동.
*   **Gemini API**: AI 태스크 태깅 및 요약보고서 생성용 LLM 연동.
*   **Pusher API**: 실시간 소켓 동기화 채널.
*   **Discord Webhook** (선택사항): 부트캠프 팀원들이 상주하는 디스코드 채널로 실시간 알림 전송.

## 3. 구조 & 권한 (Architecture & Structure)

### 🔑 회원가입 필요 여부
*   **필요 여부**: **필수**
*   **가입 방식**: **구글 소셜 로그인 및 이메일 로그인 필수 추가 (NextAuth.js)** (GitHub 단일 로그인은 비전공자의 진입 장벽이 높으므로 이메일/구글을 메인으로 삼고 GitHub는 전공자용 연동 옵션으로 제공).
*   **이유**: 팀 생성 및 팀원 매핑을 위해 안전하고 간편한 인증 체계가 마련되어야 함.

### ⚙️ 관리자 페이지 필요 여부
*   **필요 여부**: **MVP 단계 제외 (미필요)**
*   **이유**: 
    *   대학생 팀 단위의 단일 워크스페이스 구조이므로 슈퍼 관리자 기능보다는 개별 팀장이 설정할 수 있는 간단한 '팀 관리 모달'로 대체.

### 🛡️ 사용자 권한 단계
| 권한 레벨 | 주요 역할 | 주요 가능 행동 |
| :--- | :--- | :--- |
| **Team Leader (팀장/PM)** | 일정 조율 및 작업 배분 담당 | 프로젝트 설정 변경, 마일스톤 데드라인 설정, 팀원 추가/수정 |
| **Active Member (팀원)** | 개발 및 비개발 태스크 수행 주체 | 태스크 카드 등록 및 편집, 자신의 역할(개발/기획 등) 필터링, AI 피드백 요청 |
| **Mentor (멘토/평가위원)** | 프로젝트 피드백 및 모니터링 주체 | 실시간 보드 조회 및 피드백 댓글 작성 전용 (수정 권한 제한) |

## 4. 제약 사항 (Constraints)

### 📅 개발 기간 / 데드라인
*   **목표 기간**: **총 4주 (부트캠프 프로젝트 마감일 기준)**
    *   `1주차`: 기획 확정, DB 스키마 정의 및 Neon DB 셋업, UI 레이아웃 구축.
    *   `2주차`: NextAuth.js 연동 회원가입 및 실시간 칸반 보드 구축.
    *   `3주차`: AI API(Gemini) 연동 및 GitHub Webhook 테스트, Pusher 실시간 연동.
    *   `4주차`: 스타일링(CSS) 디테일 개선, 통합 테스트, 배포(Vercel) 및 발표 자료 준비.

### 💵 예산 범위
*   **예산**: **0원 (완전 무료 인프라)**
    *   모든 클라우드 리소스는 Vercel 및 Neon DB Free Tier 범위 내에서 운용.
    *   LLM API 또한 무료 제공 한도(Gemini API Free Tier 등)를 최대한 활용.

### 🔒 개인정보 수집 여부 확인
*   **수집 여부**: **최소 정보만 수집**
*   **수집 항목**: 이메일 주소, 프로필 닉네임, 프로필 이미지 URL.
*   **보안 및 규정**: OAuth 인증 시 수집 동의 체크 필수화.
